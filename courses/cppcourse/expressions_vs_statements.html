C++ Expressions vs. Statements

In C++, expressions and statements are fundamental building blocks of code.

Expressions

An expression is any piece of code that produces a value. It consists of operands and operators and can be assigned to a variable or used directly.

Examples of expressions:


int x = 5 + 3;   // 5 + 3 is an expression (evaluates to 8)
bool isTrue = (x > 5); // (x > 5) is an expression (evaluates to true)
std::cout << x;  // std::cout << x is also an expression (returns std::cout)

✔️ Expressions always have a value and a type.

Statements

A statement is a complete instruction that performs an action. It can include expressions but doesn’t always produce a value.

Types of statements:

  • Expression Statements – Expressions followed by ;.
  • Declaration Statements – Declare variables or functions.
  • Control Statements – Loops (for, while), conditionals (if), etc.
  • Compound Statements – Blocks enclosed in { }.

Examples of statements:


int a = 10;        // Declaration statement
a = a + 5;         // Expression statement
if (a > 10) {      // Control statement
    std::cout << "Greater than 10";  // Another expression statement
}

✔️ Statements execute actions but don’t necessarily return a value.

Key Differences

  • Expressions produce values, while statements perform actions.
  • Every expression can be part of a statement, but not every statement is an expression.
  • Statements often end with a semicolon (;), while expressions may appear inside statements.
