C++ Includes, Defines, and Macros

C++ provides preprocessor directives like #include, #define, and macros to manage code inclusion, define constants, and create reusable code snippets before compilation.

1. #include

The #include directive is used to include the contents of a file, such as a header file, into the source code. This is essential for using libraries and functions that are not defined within the same file.

Example:


#include <iostream> // Standard library for input/output

int main() {
    std::cout << "Hello, World!" << std::endl;
    return 0;
}

The #include directive is processed during preprocessing, bringing in the contents of iostream so you can use the std::cout object.

2. #define

The #define directive is used to define constants or macros. It replaces occurrences of the defined name with the specified value before compilation begins.

Example:


#define PI 3.14159

int main() {
    float area = PI * 10 * 10; // Using the defined constant
    std::cout << "Area: " << area << std::endl;
    return 0;
}

Here, the #define directive defines the constant PI, which is substituted with its value wherever it appears in the code.

3. Macros

Macros in C++ are essentially code snippets that can take arguments. They are similar to functions but are replaced by their expanded code during preprocessing.

Example:


#define SQUARE(x) ((x) * (x))

int main() {
    int result = SQUARE(5); // Expands to ((5) * (5))
    std::cout << "Square: " << result << std::endl;
    return 0;
}

Here, SQUARE(x) is a macro that calculates the square of a number. Macros are expanded by the preprocessor before compilation.

Key Points:

  • #include – Includes files into the source code (header files, libraries, etc.).
  • #define – Defines constants and simple macros.
  • Macros – Reusable code snippets that are expanded at preprocessing time, but they lack type safety compared to functions.
  • Macros do not perform type checking, which can sometimes lead to unexpected behavior if not used carefully.
