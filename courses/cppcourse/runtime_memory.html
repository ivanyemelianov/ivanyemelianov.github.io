C++ Runtime Memory Basics

In C++, memory is divided into different regions, each serving a specific purpose at runtime. Understanding memory allocation is crucial for efficient programming and avoiding issues like memory leaks.

Memory Regions:

  • Code Segment – Stores compiled program instructions.
  • Stack – Stores function calls, local variables, and control flow.
  • Heap – Stores dynamically allocated memory (using new/delete or malloc/free).
  • Global/Static Data – Stores global and static variables, initialized once and persists throughout the program.

Example:


#include <iostream>

int globalVar = 10; // Stored in Global/Static Data

void function() {
    int localVar = 5; // Stored in Stack
    int* dynamicVar = new int(20); // Stored in Heap
    std::cout << "Local: " << localVar << ", Dynamic: " << *dynamicVar << std::endl;
    delete dynamicVar; // Free allocated memory
}

int main() {
    function();
    return 0;
}

Key Points:

  • Stack – Fast, automatic allocation, limited size.
  • Heap – Manually managed, used for dynamic memory allocation.
  • Global/Static – Exists for the entire program duration.
  • Proper memory management is essential to prevent memory leaks.
